## Key File Locations

**Entry Points:**
- `EquiFastJoin.lua`: Single-file main code; WoW loads on addon initialization via TOC
- `EquiFastJoin.toc`: TOC manifest specifying dependencies, interface version (110200 = WoW 11.2), SavedVariables (EquiFastJoinDB)

**Configuration:**
- `EquiFastJoinDB` (SavedVariables, persisted in WTF folder): Debug flag, window position/size/scale, filter toggles (Dungeons/Raids/M+/PvP/Custom), auto-open, sound/toast prefs, frame lock
- `DEFAULTS` table (EquiFastJoin.lua, lines 11-30): Initial values for all config keys

**Core Logic:**
- State initialization (lines 5-41): Global `EFJ` table, `EFJ.State` for runtime data, `EquiFastJoinDB` for persistence
- Data gathering (lines 724-802): `GatherResults()`, `GatherQuickJoinEntries()`, `GetFreshResultInfo()`
- Processing (lines 804-835): `ProcessResultsAndMaybeShow()` determines what to display
- UI rendering (lines 335-721): `EFJ.UI:Create()`, `EFJ.UI:SetRows()`, `EFJ.UI:ShowQuickJoin()`, row height computation

**Testing:**
- No automated test files; manual testing via:
  - `/efj test` slash command → shows test banner
  - `/efj show` → display Quick Join entries
  - `/efj debug on` → enable debug output to chat

## Naming Conventions

**Files:**
- Addon filename: PascalCase `EquiFastJoin.lua` (matches addon name)
- TOC filename: PascalCase `EquiFastJoin.toc` (matches addon name)
- Media files: lowercase with descriptive suffix `efjicon*.tga`, `efjwebicon@4x.png`

**Functions:**
- Global handlers: camelCase prefixed with context: `EFJ_OpenOptions()`, `TryJoin()`, `ColorizeByClass()`
- Methods on EFJ.UI: camelCase with colon syntax: `EFJ.UI:Create()`, `EFJ.UI:ShowQuickJoin()`
- Internal helpers: camelCase: `BuildActivityText()`, `ClassifyResult()`, `ProcessResultsAndMaybeShow()`
- API wrappers: Descriptive names combining source + action: `GetActivityInfoForRes()`, `GetFreshResultInfo()`

**Variables:**
- Global state: Uppercase with underscores for constants: `ADDON_NAME`, `DEFAULTS`
- Persisted DB: Global `EquiFastJoinDB` (WoW convention)
- Local state: camelCase: `ev` (event frame), `row`, `ids`, `leaderName`
- UI elements: Descriptive + element type: `row.textLeader`, `row.iconLeader`, `row.classIcons`, `row.join` (button)
- Temporary/loop vars: Single letter: `i`, `x`, `y`, `h`, `w`, `n`

**Types/Classes:**
- No explicit type definitions (Lua); role/type inferred from name: `ClassifyResult()` returns category string, `GatherQuickJoinEntries()` returns table of entry objects
- Tables representing complex data: `EFJ.State`, `EFJ.UI`, `EFJ.Options`

## Where to Add New Code

**New Feature (e.g., sorting, role preference):**
- Primary code: Add helper functions near related logic in `EquiFastJoin.lua`
  - Filter/sort logic: Near line 723-835 (core processing)
  - UI rendering changes: Near line 335-721 (UI layer)
- Configuration: Add keys to `DEFAULTS` table (line 11); add checkbox/slider in options panel (lines 856-940)
- Options UI: Modify `EFJ.Options:Create()` function (lines 856-940)

**New Event Handler:**
- Register event in main event frame (line 839)
- Add handling in OnEvent script (line 848 in conditional chain)
- Trigger appropriate data gathering and UI updates

**New Helper Function:**
- Activity-related helpers: Place near `ClassifyResult()` (line 76) or `BuildActivityText()` (line 104)
- UI helpers: Place near row creation (line 244) or rendering functions (line 335+)
- Data access wrappers: Place near existing WoW API wrappers (lines 49-237)

**Utilities:**
- Shared helpers (color, formatting, validation): Place near top of file after state initialization (lines 49-237 "Helpers" section)
- Math/table utilities: Place near helper functions or inline if only used once

## File Organization Principles

**Single-File Monolith Rationale:**
- WoW addon convention: Minimal file overhead; simpler packaging for CurseForge
- No module system: Lua 5.1 (WoW standard) lacks module support; prefer single file over split files with manual require
- Tight coupling: UI and data gathering are interdependent; single file keeps dependencies explicit

**Section Layout (EquiFastJoin.lua):**
1. Header comment (lines 1-3)
2. Global initialization (lines 5-10)
3. Configuration defaults (lines 11-30)
4. Helper functions (lines 32-237)
5. UI layer (lines 238-722)
6. Core processing (lines 723-835)
7. Event system (lines 837-979)
8. Slash commands (lines 981-1016)

**Function Ordering:**
- Foundational helpers first (string, color, validation)
- Data access second (WoW API wrappers)
- Processing logic third (filtering, gathering)
- UI rendering last (frame creation, row population)
- Events last (tie everything together)
